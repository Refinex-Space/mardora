# 项目介绍

Markora 是一个基于 CodeMirror 6 的 Markdown 编辑器能力包。它不是一个只绑定某个 UI 框架的组件，而是一组可组合的 CodeMirror 扩展、Markdown 插件和静态预览工具。你可以把它接入 Vue 2、Vue 3、React 或任何能挂载 DOM 的前端运行时。

## 适合解决的问题

Markora 面向需要 “Markdown 原文仍然可控，但编辑体验接近所见即所得” 的场景：

- 内容管理后台、文档平台、知识库、笔记应用、博客编辑器。
- 需要标题、列表、任务列表、表格、图片、代码块、数学公式、Mermaid、HTML、Emoji 等完整 Markdown 能力的编辑器。
- 需要在编辑态和预览态保持同一套插件、语法高亮、主题和 HTML 输出的应用。
- 需要通过插件体系扩展 Markdown 语法、快捷键、装饰渲染或静态 HTML 渲染的团队。

## 核心架构

Markora 的核心入口是 `markora()`。它返回一个 CodeMirror 6 `Extension[]`，可以直接放入 `EditorState.create({ extensions })`。

```ts
import { EditorState } from "@codemirror/state";
import { EditorView } from "@codemirror/view";
import { markora } from "markora/editor";
import { allPlugins } from "markora/plugins";

const view = new EditorView({
  parent: document.getElementById("editor")!,
  state: EditorState.create({
    doc: "# Hello, Markora",
    extensions: [
      markora({
        plugins: allPlugins,
      }),
    ],
  }),
});
```

编辑器运行时由四层组成：

1. CodeMirror 6 负责文档状态、选区、事务、快捷键、滚动容器和扩展组合。
2. Lezer Markdown 负责增量解析 Markdown，并允许插件追加 `MarkdownConfig`。
3. Markora 视图插件负责把语法树转换成编辑态 decoration 和 widget。
4. Markora 插件负责具体语法、主题、快捷键、preview HTML 和 CSS。

## 包入口

Markora 发布为 `markora`，并提供多个子路径导出：

| 入口              | 用途                                                                                            |
| ----------------- | ----------------------------------------------------------------------------------------------- |
| `markora`         | 聚合导出，包含 editor、plugins、preview、lib。                                                  |
| `markora/editor`  | `markora()`、`MarkoraPlugin`、主题、i18n、slash commands、attachments、selection toolbar、TOC。 |
| `markora/plugins` | 内置插件、`essentialPlugins`、`allPlugins`。                                                    |
| `markora/preview` | 静态 HTML 渲染、CSS 生成、预览目录提取。                                                        |
| `markora/lib`     | 底层输入处理工具。                                                                              |

包同时提供 ESM、CJS 和 TypeScript 类型声明。业务侧推荐优先使用子路径导入，避免把不需要的模块带入产物。

## 需要安装的依赖

Markora 把 CodeMirror 6 作为 peer dependencies。业务项目需要安装 Markora 和以下 CodeMirror 包：

```shell
npm install markora
npm install @codemirror/commands @codemirror/lang-markdown @codemirror/language @codemirror/language-data @codemirror/state @codemirror/view
```

如果你要像 playground 一样展示 HTML/CSS 输出，还需要在应用中安装并使用对应的框架集成：

- React playground 使用 `@uiw/react-codemirror` 和 CodeMirror theme。
- Vue 2 / Vue 3 playground 直接使用 `EditorView`，不依赖额外组件封装。

## 已支持的 Markdown 能力

`allPlugins` 当前包含以下插件：

| 插件              | 能力                                                                      |
| ----------------- | ------------------------------------------------------------------------- |
| `ParagraphPlugin` | 段落 spacing 与 preview 段落渲染。                                        |
| `HeadingPlugin`   | `#` 到 `######` 标题、标题锚点、TOC 相关 preview 输出。                   |
| `InlinePlugin`    | 粗体、斜体、删除线、高亮、上标、下标、行内格式快捷键。                    |
| `LinkPlugin`      | Markdown 链接、快捷键、悬停/打开交互、preview 链接渲染。                  |
| `ListPlugin`      | 无序列表、有序列表、任务列表、嵌套列表与快捷键。                          |
| `TablePlugin`     | GFM 表格解析、编辑态装饰、preview 表格输出。                              |
| `HTMLPlugin`      | HTML block、HTML tag、注释等原生 Markdown HTML 能力。                     |
| `ImagePlugin`     | 图片语法、figure/figcaption、损坏状态、图片插入快捷键。                   |
| `MathPlugin`      | 行内数学、块级数学，基于 KaTeX 渲染。                                     |
| `MermaidPlugin`   | Mermaid 代码块识别与图表渲染。                                            |
| `CodePlugin`      | 行内代码、围栏代码块、语法高亮、标题、说明、行号、行/文本高亮、复制按钮。 |
| `QuotePlugin`     | 引用块装饰与 preview 输出。                                               |

| `EmojiPlugin`     | Emoji 语法扩展与 preview 输出。                                           |

## 编辑器能力

`markora()` 支持以下核心配置：

```ts
import type { KeyBinding } from "@codemirror/view";
import type { Extension } from "@codemirror/state";
import type { MarkdownConfig } from "@lezer/markdown";
import { markora, ThemeEnum } from "markora/editor";
import { allPlugins } from "markora/plugins";

const extensions = markora({
  theme: ThemeEnum.AUTO,
  locale: "zh-CN",
  baseStyles: true,
  plugins: allPlugins,
  markdown: [] satisfies MarkdownConfig[],
  extensions: [] satisfies Extension[],
  keymap: [] satisfies KeyBinding[],
  disableViewPlugin: false,
  defaultKeybindings: true,
  history: true,
  indentWithTab: true,
  highlightActiveLine: true,
  lineWrapping: true,
  onNodesChange: (nodes) => {
    console.log(nodes);
  },
  slashCommands: {
    enabled: true,
  },
  selectionToolbar: {
    enabled: true,
  },
  toc: {
    enabled: true,
  },
  attachments: {
    enabled: true,
    enablePaste: true,
    enableDrop: true,
    accept: {
      image: ["image/*"],
      video: ["video/*"],
      audio: ["audio/*"],
      file: ["*/*"],
    },
    async uploader(file, context) {
      const form = new FormData();
      form.append("file", file);
      form.append("kind", context.kind);

      const response = await fetch("/api/uploads", {
        method: "POST",
        body: form,
      });

      if (!response.ok) {
        throw new Error("Upload failed");
      }

      return response.json() as Promise<{
        url: string;
        name?: string;
        title?: string;
        mimeType?: string;
      }>;
    },
  },
});
```

## Preview 能力

`markora/preview` 提供静态 HTML 渲染，适合预览面板、服务端渲染、导出 HTML 或保存发布内容。

```ts
import { githubLight } from "@uiw/codemirror-theme-github";
import { ThemeEnum } from "markora/editor";
import { allPlugins } from "markora/plugins";
import { extractPreviewTocFromMarkdown, generateCSS, preview } from "markora/preview";

const html = await preview(markdown, {
  plugins: allPlugins,
  theme: ThemeEnum.LIGHT,
  syntaxTheme: githubLight,
  sanitize: true,
  wrapperTag: "article",
  wrapperClass: "markora-preview",
});

const css = generateCSS({
  plugins: allPlugins,
  theme: ThemeEnum.LIGHT,
  wrapperClass: "markora-preview",
  includeBase: true,
  syntaxTheme: githubLight,
});

const toc = extractPreviewTocFromMarkdown(markdown);
```

`preview()` 输出语义化 HTML；`generateCSS()` 输出与插件、主题、语法高亮匹配的 CSS；`extractPreviewTocFromMarkdown()` 输出可用于侧边目录的数据。

## 插件体系

自定义插件继承 `MarkoraPlugin`：

```ts
import type { SyntaxNode } from "@lezer/common";
import type { DecorationContext } from "markora/editor";
import { MarkoraPlugin } from "markora/editor";

export class MyPlugin extends MarkoraPlugin {
  readonly name = "my-plugin";
  readonly version = "1.0.0";
  readonly requiredNodes = ["Paragraph"] as const;

  buildDecorations(ctx: DecorationContext): void {
    // 在 ctx.decorations 中追加 CodeMirror Decoration。
  }

  renderToHTML(node: SyntaxNode, children: string): string | null {
    if (node.name !== "Paragraph") return null;
    return `<p data-my-plugin="true">${children}</p>`;
  }
}
```

插件可以覆盖这些能力：

- `getExtensions()`：追加 CodeMirror 扩展。
- `getMarkdownConfig()`：追加 Lezer Markdown 语法。
- `getKeymap()`：追加 CodeMirror 快捷键。
- `buildDecorations(ctx)`：贡献编辑态 decoration。
- `onRegister(ctx)`：读取 Markora 配置。
- `onViewReady(view)`：访问已创建的 `EditorView`。
- `onViewUpdate(update)`：响应文档或选区更新。
- `renderToHTML(node, children, ctx)`：贡献 preview HTML。
- `getPreviewStyles(theme, wrapperClass)`：贡献 preview CSS。

## 支持的 API 标准

Markora 接入面遵循以下标准或事实标准：

- CodeMirror 6 `Extension`、`EditorState`、`EditorView`、`KeyBinding`、transaction/update listener。
- Lezer Markdown `MarkdownConfig` 与 syntax tree 节点模型。
- ESM / CJS / TypeScript declarations / package subpath exports。
- Browser `File`、`FormData`、Clipboard paste、Drag and Drop，用于附件上传。
- CSS class scoping，用 `wrapperClass` 隔离 preview 样式。
- DOMPurify 风格的 HTML sanitize 边界，`preview({ sanitize: true })` 默认开启。

## Playground 同等效果清单

要达到当前三个 playground 的体验，需要实现这些集成点：

1. 加载 `allPlugins`，并提供插件开关。
2. 在编辑器中接入 `markora({ plugins, theme, attachments, slashCommands, selectionToolbar })`。
3. 用框架状态同步 `EditorView.updateListener` 或组件回调。
4. 维护编辑、源码、预览、输出四类模式。
5. 预览模式调用 `preview()` 和 `generateCSS()`。
6. 使用 `extractPreviewTocFromMarkdown()` 或 `toc.onTocChange` 维护目录。
7. 提供生产 uploader，把本地文件转换为可访问 URL。
8. 为用户文档做本地存储或服务端存储。
9. 随主题切换同步 Markora `ThemeEnum` 与 CodeMirror syntax theme。
10. 在卸载组件时销毁 `EditorView`，释放 `blob:` URL。

如果你只需要最小编辑器，安装依赖后创建一个 `EditorView` 即可。如果你要复制 playground 质量，需要完整处理状态、预览、目录、插件开关、附件上传和生命周期清理。
